Software & Firmware Design

The program setup that made our design functional.

Into the Guts of our Program

Our software is entirely written in C++ for arduino. Initially, we intended to use the AccelStepper library, however it proved to be finicky so we ended up going with just the standard arduino Stepper library.

We needed to allow for the limit switches and E-stops to be triggered at any time, but the .step() function from the Stepper library is blocking, so if we were to put multiple steps, the stops could only be triggered after the steppers finished moving. In order to remedy this, we put no more than 1 step per time calling .step() and then we use a for loop to run the stepper a certain number of steps while checking if switches have been triggered after each step. We used this structure throughout our code.

In order to draw diagonal lines, we have the vertical stepper take one step per loop, while the stepper that controls the rotational axis takes one step every X number loops. This results in a maximum of 2 steps in between checking limits. The higher X is, the more vertical the line is because the longer the pen travels down before moving over.
